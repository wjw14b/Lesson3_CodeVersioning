Highlight #1 Git Snapshots

Git has a different way of using its data compared to other version control systems while other systems usually store information as a list of file-based changes and keep seperate files for each change made. However Git uses its data as a set of snapshots of a file system whenever someone saves or commits a file it takes that snapshot of what the file was edited to.

Highlight #2 The Three States

The three states refer to the three main sections of a Git project. The Git directory, the working directory, and the staging area. The Git directory is where Git stores the metadata of a project. The working directory are files that are pulled out of compressed database in Git directory and placed on a disk. Finally the staging area is a file that stores info about what will go in the next commit and is refferred to as the "index".

Highlight #3 Tracking New Files

To track new files you need to use the git add command. If you trck files such as the README file you can run $ git add README, after that you run $ git status and will be able to see that your README file is now tracked and staged to be committed.
